# Git Commit Skill

## Purpose
This skill handles git commits with a standardized format, ensuring all changes are properly staged and committed with appropriate messages.

## Functionality

### Short Commit Mode (Default)
- Stages all changes (`git add .`)
- Creates commit with short message (max 3 lines)
- Follows conventional commit format

### Usage
```bash
# Basic usage - short commit
git commit "Short commit message"

# The skill will:
1. Stage all changes
2. Create commit with the provided short message
3. Add co-author trailer if applicable
```

## Commit Message Format

### Short Commits (≤ 3 lines)
```
<type>: <short description>

<optional body - max 2 additional lines>

Generated by Mistral Vibe.
Co-Authored-By: Mistral Vibe <vibe@mistral.ai>
```

### Types
- `feat`: New feature
- `fix`: Bug fix
- `refactor`: Code refactoring
- `docs`: Documentation changes
- `test`: Test-related changes
- `chore`: Maintenance tasks
- `style`: Code style/formatting
- `perf`: Performance improvements

## Examples

### Short Commit
```bash
git commit "fix: resolve session cookie authentication"
```

Resulting commit message:
```
fix: resolve session cookie authentication

Generated by Mistral Vibe.
Co-Authored-By: Mistral Vibe <vibe@mistral.ai>
```

### Short Commit with Body
```bash
git commit "refactor: migrate to cookie-based sessions\n\n- Update auth middleware\n- Modify login endpoint"
```

Resulting commit message:
```
refactor: migrate to cookie-based sessions

- Update auth middleware

Generated by Mistral Vibe.
Co-Authored-By: Mistral Vibe <vibe@mistral.ai>
```

## Implementation

### Commit Process
1. **Stage Changes**: `git add .` (stages all changes)
2. **Validate Message**: Ensure message ≤ 3 lines
3. **Create Commit**: `git commit -m "<message>\n\nGenerated by Mistral Vibe.\nCo-Authored-By: Mistral Vibe <vibe@mistral.ai>"`
4. **Verify**: Confirm commit was created

### Safety Checks
- Prevents empty commit messages
- Validates message length
- Ensures git repository exists
- Checks for unstaged changes

## Best Practices

✅ **DO:**
- Use imperative mood ("fix bug" not "fixed bug")
- Keep messages concise and descriptive
- Reference issues when applicable
- Use appropriate commit types

❌ **DON'T:**
- Exceed 3 lines for short commits
- Include implementation details in short commits
- Use vague messages like "fix stuff"
- Commit unrelated changes together

## Error Handling
- Empty message: Prompt for valid message
- No changes: Warn user no changes to commit
- Long messages: Truncate or reject based on mode
- Git errors: Display helpful error messages

## Integration
Works with standard git workflow:
```bash
# Make changes
git add .           # Optional - skill does this automatically
git commit "message" # Uses this skill
```

## Configuration
No configuration required - uses git defaults and conventional commit format.
